# 流水线与制品

> **状态：** 骨架提纲。

## 本文目标

定义 **标准流水线阶段** 与 **制品形态**，支撑可重复构建与不可变部署。

## 建议撰写要点

- **阶段**：依赖安装 → 静态检查 → 单元测试 → 构建 → 镜像/制品推送 → 部署触发。
- **缓存**：依赖缓存策略与可重现构建注意事项。
- **制品**：容器镜像 tag 策略（语义化 + Git SHA）；移动端 apk/aab/ipa 归档。
- **环境晋级**：谁有权 promote；与 [环境晋升](../platform-contracts/environments-and-api-compat.md) 对齐。
