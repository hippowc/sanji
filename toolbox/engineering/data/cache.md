# 缓存（以 Redis 为典型）

## 1. 典型用途

| 场景 | 说明 |
|------|------|
| 会话外置 | Web 无状态化 |
| 热点读缓存 | 减轻数据库压力 |
| 速率限制计数器 | 网关或应用侧滑动窗口 |
| 分布式锁 | 谨慎：需 TTL 与 fencing token 防误杀 |

## 2. 一致性心智

缓存 **不是** 第二数据源；默认 **cache-aside**：

1. 读：先缓存 → miss 则读库 → 回填。  
2. 写：更新库 → **删除或失效** 缓存（而非盲目双写）。

强一致场景慎用缓存；必要时引入同步开销更大的模式（write-through 等）。

## 3. 常见故障模式

| 问题 | 缓解 |
|------|------|
| **缓存穿透** | 对不存在 key 缓存短 TTL 空值；布隆过滤器 |
| **缓存击穿** | 热点 key 互斥重建或 singleflight |
| **缓存雪崩** | TTL 加随机抖动；多级缓存；熔断降级 |

## 4. 运维要点

- **持久化策略**：RDB/AOF 取舍；托管服务通常封装。  
- **内存上限**：淘汰策略（volatile-lru 等）与业务 SLA。  
- **集群**：分片与客户端路由复杂度上升时再引入。
