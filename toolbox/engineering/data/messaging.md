# 消息与异步

## 1. 队列 vs 流

| 类型 | 代表能力 | 典型用途 |
|------|----------|----------|
| **队列** | 点对点、消费者竞争消费 | 任务异步、削峰填谷 |
| **流** | 分区有序、可回放日志 | 事件溯源、实时管道、跨系统集成 |

托管服务在各云命名不同（SQS/SNS、Service Bus、Pub/Sub、RocketMQ、Kafka 托管版等），选型看 **顺序要求、回放需求、生态客户端**。

## 2. 投递语义

| 语义 | 说明 |
|------|------|
| 至多一次 | 可能丢，不会重复 |
| 至少一次 | 可能重复，消费端必须 **幂等** |
| 精确一次 | 端到端成本高；常在业务层用幂等逼近 |

**默认假设：至少一次 + 幂等消费。**

## 3. 幂等消费模式

- **业务幂等键**：订单号、支付流水号唯一约束。  
- **去重表**：短期窗口记录 message id。  
- **状态机**：只允许合法迁移。

## 4. 与数据库的事务边界

**Outbox 模式**：业务写入 DB 的同时写入 `outbox` 表；独立进程读取并发布消息，避免「DB 提交成功、消息未发出」双写不一致。

## 5. 背压与超时

- 消费者 **并发** 与下游承载匹配；队列深度监控告警。  
- **死信队列（DLQ）**： poison message 隔离与人工诊断。
