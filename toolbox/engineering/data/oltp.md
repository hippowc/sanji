# 关系型与 OLTP

## 1. 选型维度（托管 RDS 类）

| 维度 | 权衡 |
|------|------|
| 引擎 | PostgreSQL（扩展丰富）vs MySQL 系（生态与托管成熟度因云而异） |
| 高可用 | 多 AZ 同步/异步复制；故障切换 RTO |
| 规格 | CPU/内存/IOPS；初期宁小勿过大 + 观测驱动扩容 |
| 网络 | 私网访问；安全组最小放行 |

## 2. 连接管理

- **连接池**：应用侧池大小 × 实例副本数 **不得超过** 数据库 `max_connections` 的安全阈值。  
- **短连接风暴**：Serverless 形态注意 **RDS Proxy / PgBouncer** 类池化。  
- **超时**：语句超时、事务超时，防止拖垮连接池。

## 3. Schema 与迁移

使用迁移工具（见 [迁移与发布](../cicd/db-migrations-and-release.md)）；**禁止**仅在生产手工 `ALTER` 而不回补脚本。

## 4. 备份与高可用

- **自动备份**：保留期满足业务 RPO；定期 **恢复演练**。  
- **只读副本**：报表类查询卸载；注意复制延迟对读一致性的影响。

## 5. 安全

- **最小账号**：应用账号仅 DML 所需；DDL 由迁移 Job 专用账号执行（可选拆分）。  
- **加密**：静态加密（KMS）与传输 TLS；密钥轮换策略。
