# 桌面客户端

## 1. 技术选型维度

| 方案 | 优点 | 代价 |
|------|------|------|
| **Electron** | 生态成熟、Web 技术栈复用 | 包体积大、内存占用高 |
| **Tauri** | 体积小、Rust 后端安全面可控 | 生态较新、部分原生能力需自行封装 |
| **.NET MAUI / Qt / 原生** | 性能与系统集成好 | 技能栈独立、交付成本高 |

选型公式：**团队 Web 资产复用度 × 体积与内存约束 × 安全合规要求**。

## 2. 自动更新

- **差分更新**：减少带宽；校验签名 **必备**。  
- **失败降级**：下载失败回滚上一版本；避免「半更新」状态。  
- **渠道**：正式通道与 Beta 通道分离。

## 3. 签名与公证

- **Windows**：代码签名证书；SmartScreen 信誉累积。  
- **macOS**：Apple 公证（Notarization）；Gatekeeper。  
- CI 中 **密钥在线时间短**：使用 HSM 或云签名服务（按合规选型）。

## 4. 安全

- **禁用 Node integration + 上下文隔离**（Electron）；`preload` 最小暴露 API。  
- **深度链接**：校验 scheme 与白名单。  
- **自动更新服务器**：防劫持（HTTPS + 签名验证）。

## 5. 观测与隐私

- **崩溃上报**：Sentry 等；默认 **脱敏** PII。  
- **日志**：本地轮转；上传需用户同意（视司法辖区）。
