# 网络拓扑与暴露面

## 1. 典型三层逻辑

```
Internet
   │
   ▼
[公网负载均衡 / API 网关] ──► 仅 443（及管理堡垒，若需要）
   │
   ▼
[应用子网] 无状态计算实例 / Pod
   │
   ▼
[数据子网] RDS、Redis、消息 —— 仅私网 + 安全组来源=应用安全组
```

## 2. 子网与安全组/防火墙

| 实践 | 说明 |
|------|------|
| **默认拒绝** | 仅放行必要端口与源 |
| **分层 SG** | 应用层 vs 数据层分离；引用安全组 ID 而非超大 CIDR |
| **出站限制** | 生产实例默认允许 NAT 访问外网需评估；敏感环境走代理白名单 |

## 3. 管理访问

- **避免** 全体开发 SSH 直连生产数据库。  
- 采用 **堡垒机**、**Session Manager 类** 会话管理、或 **临时凭证 + 审计录屏**（按合规）。

## 4. 出站依赖

调用第三方 API（支付、推送）：固定 egress IP（NAT 网关）便于对方白名单。

## 5. DNS 与私有解析

- 内部服务使用 **私有 Hosted Zone**；避免依赖公网 DNS 暴露内部主机名。
