# 链路追踪

## 1. 价值

- 分解 **跨服务延迟**。  
- 发现 **意外依赖** 与 N+1 调用。  
- 与日志通过 `trace_id` 关联。

## 2. 何时引入

| 阶段 | 建议 |
|------|------|
| 单体或极少服务 | 可仅用日志 + 指标 |
| ≥3 服务协作且延迟敏感 | 引入采样 tracing |
| 大规模网格 | 完整策略 + tail-based 采样（进阶） |

## 3. 采样

- **头部采样**：入口处随机或自适应决定追踪与否；实现简单。  
- **尾部采样**（进阶）：保留异常或慢请求全链路；存储成本高但诊断价值大。

## 4. 传播标准

采用 **W3C Trace Context**（`traceparent` header）便于跨语言和网关。

## 5. 成本治理

- Span 数量与自定义标签克制；业务字段勿无限膨胀。  
- 保留期 **短于** 日志全量保留常见。
