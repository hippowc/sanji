# 总纲

> **状态：** 骨架提纲。

## 本文目标

说明本知识体系的 **适用范围、公有云假设、非目标**，并指向最小生产底线与术语表。

## 建议撰写要点

- **读者画像**：能在公有云控制台与 IaC 之间完成基础操作的开发者或 Tech Lead。
- **公有云假设**：默认使用托管 VPC、托管负载均衡、托管数据库与密钥管理；不写单一厂商独占步骤时可采用「能力 × 典型托管服务类型」表述。
- **非目标**：不设大规模数据平台、不设完整 FinOps 专著、不设特定语言源码教学全文。
- **与 Well-Architected 对齐**：可在附录用表格对照「可靠性、安全、性能、成本、运维」作为自检清单（待写）。
- **版本声明**：文首注明知识体系基准年份与复审频率建议。
