## 设计原则（贯穿全书）

1. **能力优先于品牌**：先明确要解决的能力（身份、密钥、备份、发布），再映射到具体托管服务或开源组件。
2. **默认拒绝，最小暴露**：公网仅暴露必要入口；数据面与管理面分离。
3. **不可变制品与可回滚**：发布单元可追溯；失败有权回到上一已知良好版本。
4. **契约驱动多端**：移动端长周期驻留，API 与配置变更必须考虑 **向后兼容** 与 **契约测试**。
5. **可观测是组成部分**：不是上线后再补；最小生产应同时具备日志、核心指标与基础告警。
