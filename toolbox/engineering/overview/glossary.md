# 术语表

下列条目在全书中多次出现，此处统一 **中文表述**；英文保留便于检索与对齐官方文档。

## 可靠性与目标

| 术语 | 说明 |
|------|------|
| **SLI（Service Level Indicator）** | 服务水平的 **测量指标**，如可用性比例、P99 延迟。 |
| **SLO（Service Level Objective）** | 对 SLI 的 **内部目标**，如月度可用性 ≥ 99.9%。 |
| **SLA（Service Level Agreement）** | 对用户的 **合同级承诺**，违约常伴随赔偿；比 SLO 更严格、更正式。 |
| **错误预算（Error Budget）** | `1 - SLO` 所允许的不可靠「额度」；用于权衡发布频率与稳定性投入。 |
| **RPO（Recovery Point Objective）** | 灾难发生后 **最多可接受丢失的数据时间窗口**（备份粒度决定）。 |
| **RTO（Recovery Time Objective）** | 灾难发生后 **最多可接受的服务中断时间**（恢复流程决定）。 |

## 架构与发布

| 术语 | 说明 |
|------|------|
| **不可变基础设施** | 运行时镜像/制品版本固定，变更通过 **替换实例** 而非原地修改。 |
| **灰度发布 / 金丝雀** | 先让小比例流量使用新版本，验证后再扩大。 |
| **蓝绿部署** | 两套环境切换流量；回滚快，资源占用高。 |
| **滚动发布** | 逐批替换实例；需配合健康检查与就绪探针。 |
| **BFF（Backend for Frontend）** | 面向特定客户端（Web/移动）的 **聚合与适配层**，减少往返与协议摩擦。 |

## 身份与密钥

| 术语 | 说明 |
|------|------|
| **IAM** | 身份与访问管理：谁（主体）能对哪些资源执行哪些动作。 |
| **OIDC** | 基于 OAuth 2.0 的身份层；常用于 CI 与云平台之间的 **联邦登录**。 |
| **Workload Identity / IRSA 等** | 运行中的工作负载 **自带身份**，向云平台换取短期凭证，避免静态密钥。 |

## 可观测性

| 术语 | 说明 |
|------|------|
| **日志（Logs）** | 离散事件记录，适合排错与审计。 |
| **指标（Metrics）** | 时间序列聚合数据，适合告警与容量。 |
| **链路追踪（Tracing）** | 请求在分布式系统中的路径与耗时分解。 |
| **RED 方法** | Rate、Errors、Duration：面向 **请求驱动服务** 的黄金信号。 |
| **USE 方法** | Utilization、Saturation、Errors：面向 **资源** 的利用率诊断。 |

## 数据与消息

| 术语 | 说明 |
|------|------|
| **OLTP** | 在线事务处理：面向日常业务读写，强调一致性与低延迟。 |
| **幂等（Idempotency）** | 同一操作执行多次与执行一次 **效果相同**；重试与消息消费的关键。 |
| **至少一次 / 至多一次 / 精确一次** | 消息投递语义；分布式系统中「精确一次」成本高，常在业务层幂等实现。 |
