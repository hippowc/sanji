# 最小生产（Minimal Production）定义

**最小生产** 指：在真实流量与真实数据下可持续运行、具备基本故障应对与发布纪律的环境。它与「演示环境」的差别不在于规模，而在于 **是否具备工程底线**。

下文给出建议的 **底线清单** 与 **可延后项**；团队可按合规要求加码，不建议随意删减底线。

## 一、建议底线（能力视角）

### 1. 可用性与发布

| 能力 | 最低要求 | 说明 |
|------|----------|------|
| 健康检查 | 就绪探针与存活探针（或平台等价能力） | 未就绪实例不接流量；僵死进程可被替换 |
| 多实例或自动恢复 | 至少具备其一 | 单实例时须有进程/容器自动拉起；有状态组件依赖托管高可用 |
| 不可变制品 | 镜像或发布包版本可追溯 | 禁止 SSH 登入后改代码即生效 |
| 回滚 | 15～30 分钟内可恢复到上一版本 | 含应用与配置；数据库回滚策略单独约定 |

### 2. 数据

| 能力 | 最低要求 |
|------|----------|
| OLTP 备份 | 托管自动备份开启；保留期与业务 RPO 对齐并文档化 |
| 恢复演练 | 至少做过一次「从备份恢复到非生产」并记录耗时 |
| 迁移纪律 | Schema 变更走迁移工具或脚本；与发布顺序在 Runbook 中有说明 |

### 3. 安全与网络

| 能力 | 最低要求 |
|------|----------|
| 传输加密 | 对公网与用户侧强制 HTTPS（TLS） |
| 密钥 | 生产密钥不进源码仓库；使用云平台密钥管理或等价方案 |
| 身份 | 运行时与 CI 使用 **工作负载身份 / OIDC** 等短期凭证模式，避免共享长期 Access Key |
| 网络 | 数据库、缓存等默认仅私网可达；安全组/防火墙默认拒绝 |

### 4. 可观测与响应

| 能力 | 最低要求 |
|------|----------|
| 日志 | 应用日志结构化（建议 JSON）；集中可查；敏感字段脱敏 |
| 指标 | 请求量、错误率、延迟分位数（或平台等价 golden signal） |
| 告警 | 对用户有影响的症状类告警；对接值班或通知渠道 |
| Runbook | 至少覆盖：发布失败回滚、数据库连接失败、错误率飙升初判 |

## 二、可延后（需标注风险）

以下能力在规模较小时可延后，但应在架构评审中 **明确「未具备」与补偿措施**：

- 多可用区主动流量切换与定期演练
- 全链路追踪的高采样与长期存储
- 完整服务网格与细粒度 mTLS 全链
- 高级 WAF 规则与 Bot 管理
- 成本分摊标签体系与部门级预算治理

## 三、最小拓扑（逻辑示意）

```
用户 / 客户端
    │ HTTPS
    ▼
[托管负载均衡 / API 网关]
    │
    ▼
[计算层：托管容器 / Serverless 容器 / VM 上的编排]
    │        │
    │        └──► [托管密钥]、[托管配置]
    ▼
[私网]
    ├── 托管 OLTP（RDS 等）
    ├── 托管缓存（Redis 等，按需）
    └── 对象存储（静态资源 / 附件，按需）
```

日志与指标由平台 Agent 或托管集成采集至统一查询与告警。

## 四、自检表（上线前快速过一遍）

- [ ] 生产与开发/测试 **账号或项目** 隔离，权限最小化  
- [ ] 数据库 **无公网暴露** 或已严格白名单并审计  
- [ ] **备份** 开启且保留策略明确；做过恢复演练记录  
- [ ] **密钥** Rotation 策略明确；CI 无长期云密钥  
- [ ] **发布与回滚** 文档化；Feature Flag（如有）有Kill Switch  
- [ ] **告警** 能到达责任人；非工作时间有升级路径  
- [ ] **移动端 / 旧客户端** API 兼容策略已写在契约或 ADR
