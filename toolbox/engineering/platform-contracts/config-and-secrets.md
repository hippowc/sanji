# 配置、密钥与十二因子实践

## 1. 配置分级

建议将配置分为三类，分别约束来源与变更频率：

| 类型 | 示例 | 存储与下发 |
|------|------|------------|
| **内置默认** | 超时默认值、功能开关默认关闭 | 代码或镜像内；仅允许非敏感默认值 |
| **环境配置** | 依赖的外部 endpoint、池大小、日志级别 | 运行时注入（环境变量、托管配置、挂载文件） |
| **密钥与证书** | 数据库口令、API Token、TLS 私钥 | **仅** 云平台密钥管理或等价 Secret 存储；按需挂载或按需 API 获取 |

**原则：** 密钥永不写入镜像层与 Git 历史；环境配置也不应硬编码在编译产物中导致跨环境复用错误。

## 2. 与十二因子（Twelve-Factor）的对齐要点

十二因子是经典云原生应用约束，下列条目与最小生产强相关：

1. **基准代码**：一份代码库多环境部署；环境差异靠配置而非分叉仓库。  
2. **依赖**：显式声明；构建可重现（锁文件）。  
3. **配置**：严格外部化（见上表）。  
4. **后端服务**：把数据库、缓存等当作 **可替换资源**，通过 URL/连接串绑定。  
5. **构建、发布、运行**：构建产物 immutable；运行阶段不改代码。  
6. **进程**：无状态；会话外置到缓存或令牌。  
7. **端口绑定**：应用自给导出端口（容器平台再映射）。  
8. **并发**：水平扩展而非单靠纵向巨型单机。  
9. **易处理**：快速启动、优雅退出（SIGTERM）。  
10. **开发/生产等价**：尽量缩小拓扑差异（staging 逼近 prod）。  
11. **日志**：当作 **事件流** 输出 stdout/stderr，由平台采集。  
12. **管理任务**：与常驻进程同一代码版本与配置（迁移 Job、批处理）。

不必教条主义逐字背诵；评审时可用上述条目做 checklist。

## 3. 密钥操作清单

- **轮换**：数据库账号、签名密钥、第三方 API Key 应有轮换周期与负责人。  
- **最小权限**：应用账号仅所需 schema/读写权限；禁止 root 直连应用。  
- **审计**：谁在何时读取了哪类 Secret（依赖云平台审计能力）。  
- **本地开发**：`.env` 列入 `.gitignore`；提供 `.env.example` 仅含占位符与说明。  
- **CI**：使用 OIDC 换取短期云凭证（详见 [密钥与身份](../cicd/secrets-identity.md)）。

## 4. 日志中的关联字段（契约）

建议在每条请求日志中携带（或由网关注入）：

- `request_id`：网关或边缘生成的唯一 ID。  
- `trace_id`：若启用分布式追踪，与 Span 根一致。  
- `user_id` 或匿名标识：业务相关；注意合规与脱敏。

便于将 **日志 ↔ 指标 ↔ 链路** 串联。

## 5. 常见反模式

| 反模式 | 风险 |
|--------|------|
| 前端 bundle 内含生产 API 密钥 | 密钥泄露；滥用配额 |
| 日志打印完整 SQL 含敏感参数 | 违规与撞库风险 |
| 生产连接串提交到私有仓库但未加密管控 | 泄露面扩大 |
| 所有环境共用一套数据库 | 测试脚本误删生产数据 |
