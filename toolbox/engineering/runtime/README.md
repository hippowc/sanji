# 运行时：部署形态与演进

运行时回答：**工作负载放在何种计算抽象上**、如何伸缩、如何接入流量与配置。本卷按 **从小到大** 描述公有云常见路径，并单独讨论 **托管 Kubernetes** 的引入时机。

## 原则

- **匹配团队运维半径**：抽象层级越高，自定义空间越小，运维负担越低。  
- **与服务网格解耦**：多数最小生产不需要网格；需要时再引入。
