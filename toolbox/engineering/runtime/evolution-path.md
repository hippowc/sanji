# 从小到大演进路径

> **状态：** 骨架提纲。

## 本文目标

给出公有云上常见的 **计算承载形态递进**，并说明何时进入下一阶段。

## 建议撰写要点

### 典型阶梯（名称用能力描述，举例可用多云对标）

1. **单实例托管计算**：托管容器单服务 / Serverless 容器；托管负载均衡；适用 PoC 与小流量。
2. **多实例 + 弹性**：健康检查、滚动升级；会话外置（缓存或 JWT）。
3. **托管 Kubernetes**：多服务、统一流量入口、配置与密钥规模化；运维成本上升。
4. **再进阶**（可选）：多集群、混合云——标明超出最小生产默认范围。

### 每张阶梯必备

- **何时升级**：延迟、成本、团队人数、发布频率等触发条件。
- **最小配套**：日志、指标、备份是否已就绪（链到其它卷）。
