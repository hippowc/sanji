# 场景化参考技术栈（组合选型）

本目录与 [`oss-landscape.md`](../oss-landscape.md) 的差别：

| 维度 | `oss-landscape` | `stacks/`（本目录） |
|------|-----------------|---------------------|
| 组织方式 | 按 **能力类目** 罗列常用开源 | 按 **端到端场景** 给出「核心系统 + 分层组合」 |
| 读者收益 | 快速查「这类问题常用哪些项目」 | 理解 **为何这样叠积木**、与 **一体化发行版** 的对应关系 |
| 典型类比 | 零件手册 | **参考架构 / Recipe** |

每条「栈」推荐采用统一骨架（下文模板），便于你后续自己追加场景（如「实时协作」「电商订单」「IoT 接入」等）。

---

## 撰写模板（复制到新文件）

1. **场景定义**：解决什么问题、典型用户是谁、**非目标**（不解决什么）。  
2. **核心系统（Core）**：单一事实来源或最强约束的那个组件（常为某种数据库或事件日志）。  
3. **能力分层**：围绕核心向外扩展——例如 访问层、身份、实时、对象存储、函数计算、运维面。  
4. **一体化发行版**：市场上是否已有「打包好的组合」（如 Supabase、Hasura Cloud、Appwrite），与自建的取舍。  
5. **最小生产检查**：与该场景相关的安全、备份、多租户、配额（若适用）。  
6. **与其他卷的链接**：数据、运行时、安全、可观测等。
